## 1. Кнопки перелистывания (scroll up/down)

**Цель:** Показывать кнопки только когда контент не помещается на одном экране; по клику — скролл в самое начало/конец страницы.

**Текущее поведение:**

- `site/script.js` ~1107–1129: `initScrollNavButtons()` — видимость по порогу 20px (`canScrollUp` / `canScrollDown`).
- Клик: `scrollBy({ top: ±320 })` — пошаговый скролл.

**Изменения:**

- **Условие показа:** кнопки видимы только если `scrollHeight > innerHeight` (контент реально не помещается). Убрать порог 20px для «есть ли куда скроллить» или оставить минимальный порог только для плавности (опционально).
- **Действие кнопок:**
    - «Вверх» — `window.scrollTo({ top: 0, behavior: 'smooth' })`.
    - «Вниз» — `window.scrollTo({ top: document.documentElement.scrollHeight - window.innerHeight, behavior: 'smooth' })`.
- При `resize` пересчитывать видимость.

**Файлы:** `site/script.js` (функция `initScrollNavButtons`), при необходимости минимальные правки в `site/index.html` (aria/классы контейнера).

## 9. Переключение светлой/тёмной темы

**Цель:** Кнопка переключения темы должна корректно менять тему.

**Текущее:**

- `site/js/ui/theme-toggle.js`: по клику вызывается `deps.setTheme(nextTheme)` и `deps.saveUserPreferences()`.
- Тема применяется в `site/js/components/theme.js` — `setTheme(mode)` переключает `document.documentElement.classList.toggle('dark', ...)` и `dataset.theme`.
- Зависимости theme-toggle задаются в `site/script.js` 4660: `setTheme`, `State`, `saveUserPreferences` и др.

**Гипотеза проблемы:**

- Либо `State.userPreferences` не инициализирован к моменту клика.
- Либо `setTheme` не тот (не из theme.js).
- Либо порядок загрузки скриптов: theme применяется до того, как Tailwind/стили учитывают класс `dark`.
- Либо `document.documentElement.dataset.theme` не совпадает с тем, что читает приложение (например, читается `data-theme` в другом месте).

**Изменения (диагностика и исправление):**

- Убедиться, что при загрузке страницы вызывается `applyInitialUISettings` / загрузка userPreferences и применение темы.
- Проверить, что `setThemeToggleDependencies` получает именно экспорт `setTheme` из `theme.js`.
- Добавить при необходимости защиту: если `userPreferences` ещё нет — применить тему только к DOM и отложить сохранение до готовности State.
- Проверить конфиг Tailwind: `darkMode: 'class'` — класс `dark` на `html`/`documentElement` должен переключать тему.

**Файлы:** `site/js/ui/theme-toggle.js`, `site/js/components/theme.js`, `site/script.js` (инициализация и зависимости), при необходимости порядок инициализации в `app-init.js`.

## 11. Ошибка консоли: addEscapeHandler function not found

**Цель:** Убрать предупреждение `[showOrganizeFoldersModal] addEscapeHandler function not found`.

**Причина:**

- В `site/js/components/bookmarks.js` зависимость `addEscapeHandler` приходит через `setBookmarksDependencies(deps)`.
- В `site/script.js` функция `addEscapeHandler` определена (строка 892), передаётся в Bookmarks (1223, 4293 и др.), но **не экспортируется в `window`**.
- В `site/js/entry.js` для связи модулей используется `window.addEscapeHandler` — если entry выполняется до того, как script.js присвоил `window.addEscapeHandler`, зависимость будет `undefined`.
- В script.js в конце файла есть присвоения в `window` (showOrganizeFoldersModal и др.), но **не найдено присвоения `window.addEscapeHandler = addEscapeHandler`**.

**Изменения:**

- В `site/script.js` в блоке присвоений в `window` (после строки ~4919) добавить:  
  `if (typeof addEscapeHandler === 'function') window.addEscapeHandler = addEscapeHandler;`
- Убедиться, что порядок загрузки скриптов такой, что к моменту вызова `setDependencies` в entry.js все эти функции уже в `window`.

**Файлы:** `site/script.js`.

## 13. Системные уведомления таймера на всех платформах

**Цель:** Убедиться, что на всех поддерживаемых платформах (Windows, macOS, Linux, мобильные при наличии) таймер может показывать системное уведомление о завершении времени.

**Текущее:**

- `site/js/features/timer.js`: `requestAppNotificationPermission()`, `showAppNotification()` — используется Web API `Notification`.
- При отказе или недоступности — fallback на `alert()`.

**Изменения:**

- Оставить использование стандартного `Notification` API; проверить, что при первом запуске таймера или при первом истечении вызывается запрос разрешения (если ещё не выдан).
- В коде добавить комментарии или лог для платформ, где Notification может не работать (например старые браузеры, некоторые in-app WebView).
- Документировать в коде или в описании: для десктопа уведомления работают в современных браузерах; для PWA на мобильных — зависит от поддержки браузером.
- Реализацию не менять кардинально — только убедиться в вызове запроса разрешения (например при первом старте таймера) и в fallback на alert.

**Файлы:** `site/js/features/timer.js` (инициализация и `handleTimerEnd`).

## 15. Таймер: шаг +/- и предупреждение за 30 секунд

**Цель:**

- Кнопки +/-: по умолчанию ±5 сек; с Ctrl — ±10 сек; с Ctrl+Shift — ±30 сек.
- За 30 секунд до окончания — цифры и элементы таймера ярко-красные, пульсация, явное предупреждение.

**Текущее:**

- В `site/js/features/timer.js` 894–906: кнопки вызывают `adjustTimerDuration(±30)` по умолчанию; в hotkeys 228–241: Ctrl+Alt+Plus/Minus передают ±5.
- Текущая логика в коде: кнопки ±30, в hotkeys ±5 — нужно унифицировать и расширить модификаторами.
- В `updateTimerDisplay()` нет класса/состояния «осталось ≤30 сек».

**Изменения:**

- **Кнопки +/- в UI:**
    - При клике без модификаторов: `adjustTimerDuration(5)` или `-5`.
    - При Ctrl: ±10 сек.
    - При Ctrl+Shift: ±30 сек.
    - В `hotkeys-handler.js` для Ctrl+Alt+Plus/Minus оставить ±5 по умолчанию; при необходимости добавить варианты с Ctrl/Shift для ±10/±30.
- **Отображение за 30 секунд:**
    - В `updateTimerDisplay()` (или в месте, где задаются классы элементам таймера): если `timeLeftVisual <= 30` и таймер запущен — добавить на контейнер/сегменты таймера класс (например `timer-warning` или `timer-urgent`).
    - В CSS: для этого класса — цвет ярко-красный (например `#dc2626` или переменная), анимация `animation: pulse` (или кастомная пульсация opacity/scale).
    - Убедиться, что пульсация не слишком агрессивная для доступности (можно ограничить по контрасту и частоте).

**Файлы:** `site/js/features/timer.js` (обработчики кнопок, `adjustTimerDuration`, `updateTimerDisplay`), `site/js/ui/hotkeys-handler.js`, `site/css/styles.css`.
